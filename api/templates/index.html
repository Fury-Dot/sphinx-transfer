ABOUT

How SecureTransfer works

Encryption Process:

1. A 16-byte random salt is generated

2. PBKDF2-HMAC-SHA256 derives the AES key (100,000 iterations)

3. AES-CTR encrypts the file with a random 16-byte nonce

4. HMAC-SHA256 creates an integrity tag over Salt + Nonce + Ciphertext

5. Output: [ Salt(16) | Nonce(16) | Ciphertext | HMAC(32) ]

Decryption Process:

1. Extract Salt and Nonce from the bundle header

2. Derive the AES key using PBKDF2 with the same password

3. Verify HMAC integrity tag — reject if mismatch

4. AES-CTR decrypt the ciphertext to recover the original file

Security Notes:

• Your password never leaves the server in plaintext

• Files are processed in memory and immediately deleted

• HMAC verification ensures tamper detection

• 100,000 PBKDF2 iterations protect against brute-force
